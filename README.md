# 📊 Customer Support Ticket Analyzer

## 📌 Overview
This project is a Python-based **Customer Support Ticket Analysis System** designed to store, clean, analyze, and extract insights from customer service tickets.  
It helps organizations understand customer issues, improve response time, and enhance service quality through data-driven insights.

---

## 🎯 Objective (Problem Statement)
Customer support teams often deal with large volumes of unstructured ticket data, making it difficult to identify patterns, prioritize issues, and improve service quality.  
This project aims to transform raw ticket data into meaningful insights for better decision-making.

---

## 🔍 This Project Focuses On
- Managing ticket data (ID, customer name, issue, priority)
- Cleaning and standardizing text data
- Performing descriptive, diagnostic, predictive, and prescriptive analytics
- Extracting insights from customer feedback

---

## ⚙️ Tools & Technologies
- Python 🐍
- Pandas (Data Handling)
- Basic NLP (Text Cleaning & Keyword Analysis)
- Dictionaries & Lists (Data Storage)

---

## 📁 Project Structure
- **Data Storage** → Dictionary-based ticket system  
- **Data Cleaning** → Text preprocessing (lowercase, punctuation removal, normalization)  
- **Analysis Modules**:
  - Descriptive Analytics
  - Diagnostic Analytics
  - Predictive Analytics
  - Prescriptive Analytics  

---

## 📈 Analytics Performed

### 🔹 Descriptive Analytics
- Ticket count and priority distribution
- Keyword frequency analysis

### 🔹 Diagnostic Analytics
- Identified root causes like slow response and poor service

### 🔹 Predictive Analytics
- Forecasts future ticket trends based on historical data

### 🔹 Prescriptive Analytics
- Suggested improvements:
  - Faster response times
  - Better training
  - Automation for common issues
  - Escalation rules for critical tickets
